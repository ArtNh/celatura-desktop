# Celatura CSS 诊断与 IDE 指令拓展规范手记

> 本文档记录对 `src/app/globals.css` 中 `@tailwind` 未识别警告问题的分析诊断、修复策略及配置实施过程。

---

## 诊断背景与问题描述

### 问题记录
- **问题文件**：`src/app/globals.css`
- **警告内容**：`Unknown at rule @tailwind` (Line 1, 2, 3)
- **根本原因**：IDE (VS Code / Standard CSS Linter) 自带的语法分析器原生不支持 Tailwind CSS 特有的 `@tailwind`、`@apply` 和 `@layer` 等自定义 CSS At-Rules，导致编辑器触发非致命的 Linting 警告。

---

## 修复实施细节

### [2026-07-23 00:48:15] IDE 配置文件建立

```json
// .vscode/settings.json
{
  "css.lint.unknownAtRules": "ignore",
  "scss.lint.unknownAtRules": "ignore",
  "less.lint.unknownAtRules": "ignore"
}
```

> **效果说明**：
> 1. 配置规则使标准 CSS / SCSS / LESS Linter 在识别到 Tailwind 特有指令时静默放行。
> 2. 避免假阳性警告（False Positives）干扰日常代码编写体验。
> 3. 不影响 PostCSS 及 Tailwind 编译期的语法分析与 CSS 生成。
