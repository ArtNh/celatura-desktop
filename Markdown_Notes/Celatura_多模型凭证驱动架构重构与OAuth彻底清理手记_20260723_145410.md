# Celatura 多模型凭证驱动架构重构与 OAuth 彻底清理手记

### [2026-07-23 14:54:10] 多模型凭证驱动架构全闭环重构完成

---

## 重构目标回顾

彻底移除原有复杂的 Google OAuth2 授权卡片拦截、深层链接 `celatura://` 监听与临时 HTTP 回调服务逻辑，重构为商业级“多模型凭证驱动”架构。支持 **Gemini**, **DeepSeek**, **Custom OpenAI-compatible** 模型凭证配置，并实现系统 `GEMINI_API_KEY` 环境变量自动感应与无感点亮。

---

## 核心实现与代码变更

1. **彻底移除 OAuth 登录拦截**
   - **`src/app/page.tsx`**：移除了 `AuthCard` 登录拦截门槛与 `isAuthenticated` 拦截状态，软件启动后无缝渲染双栏智能体工作台（`Sidebar` + `ChatStreamView`）。
   - **`src/components/AuthCard.tsx`**：从代码库中物理彻底清理。

2. **新增极简暗黑毛玻璃凭证设置面板**
   - **`src/components/SettingsModal.tsx`**：全新暗黑毛玻璃风格凭证配置弹窗。支持独立的 API Key 输入、实时密码明密文切换、系统 `GEMINI_API_KEY` 环境变量识别提示与状态灯。
   - **`src/components/Sidebar.tsx`**：底部设置齿轮绑定触发 `SettingsModal` 弹窗；替换原 OAuth 卡片为“多模型凭证就绪状态指示器”。

3. **盘活与增强大模型后端管道**
   - **`src-tauri/src/lib.rs`**：完全清理了繁重的 OAuth/DeepLink 代码；新增 `ModelConfig` 结构体及 `load_model_config`, `save_model_config`, `check_api_key_status` Tauri 命令；改造 `execute_gemini_task` 命令，支持向子进程环境变量动态注入用户指定的 API Key 和模型参数。
   - **`src/components/ChatStreamView.tsx`**：顶部 Header 引入模型切换下拉列表（Gemini 1.5 Pro / Flash / DeepSeek V3 等），在触发对话任务时将选定的模型标识和配置透传给 Rust 后端。
