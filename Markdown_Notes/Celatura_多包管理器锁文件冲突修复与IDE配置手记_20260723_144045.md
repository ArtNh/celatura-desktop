# Celatura 多包管理器锁文件冲突修复与 IDE 配置手记

### [2026-07-23 14:40:45] 修复 VS Code 多锁文件警告提示

---

## 问题背景与诊断

> VS Code 的 `npm` 支持插件在检测到工作区根目录同时存在多个包管理器锁文件（如 `package-lock.json` 与 `pnpm-lock.yaml`）且配置项 `"npm.packageManager"` 设为 `"auto"` 时，会触发冲突提醒通知。

---

## 实施解决方案

分两步完成修复与规范收敛：

1. **显式配置首选包管理器**  
   在项目根目录 `.vscode/settings.json` 中配置 `"npm.packageManager": "npm"`，明确声明当前项目使用 `npm` 作为标准依赖管理工具。

2. **清理冗余锁文件**  
   清理并从 Git 版本库中移除多余的 `pnpm-lock.yaml` 文件，保留统一的 `package-lock.json`，确保依赖锁定机制的一致性与规范化。

---

## 变更文件汇总

- **[.vscode/settings.json](file:///d:/AI_Tools/Celatura-desktop/.vscode/settings.json)**：新增 `"npm.packageManager": "npm"`
- **`pnpm-lock.yaml`**：全自动清理删除
