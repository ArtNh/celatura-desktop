# Celatura Google 用户自定义客户端凭证配置功能实现手记

> 本文档记录 Celatura Desktop 移除代码中硬编码的静态 OAuth 凭证，重构为支持用户本地自定义输入、持久化存储与动态加载 Google Client ID 及 Client Secret 的开源工程升级过程。

---

## 需求背景与安全设计

### 1. 开源安全性与解耦要求
在开源客户端项目中，将任何秘钥凭证写死在二进制或静态代码中均属于安全隐患。提供本地自定义凭证面板，不仅允许用户使用自己的 Google Cloud 项目防配额限制，也提升了代码库的开源专业度。

### 2. 存储与加载机制
- **数据结构**：定义 `ClientCredentials` 结构（包含 `client_id` 与 `client_secret`）。
- **文件隔离**：凭证配置独立持久化存放于应用配置目录下的 `client_credentials.json` 文件中。

---

## 核心实现说明

### 1. Rust 后端动态凭证管理 ([src-tauri/src/lib.rs](file:///d:/AI_Tools/Celatura-desktop/src-tauri/src/lib.rs))
- **`save_client_credentials` Command**：将用户输入的 Client ID 和 Client Secret 持久化存储至 `client_credentials.json`。
- **`load_client_credentials` Command**：读取本地存储的凭证配置。
- **`start_google_oauth` 命令增强**：动态调用 `load_client_credentials`。若未检出有效凭证且前端未显式传参，弹出优雅的技术提示指导用户设置。

---

### 2. 前端 AuthCard 设置面板 ([src/components/AuthCard.tsx](file:///d:/AI_Tools/Celatura-desktop/src/components/AuthCard.tsx))
- **初始化检测**：组件挂载时调用 `load_client_credentials`。如已有配置，自动填入并收起面板；如无配置，自动展示输入区。
- **黑曜石暗黑设置区**：
  * 提供 `Google Client ID` 输入框。
  * 提供 `Google Client Secret` 密码隐蔽与 Eye 图标切换输入框。
  * 提供“保存 OAuth 凭证到本地”按钮，支持绿光动画与反馈提示。
