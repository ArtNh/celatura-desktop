# 贡献指南

> **声明**：本项目为 ArtNh 的个人数字化 portfolio/资产库。若有改进建议，请通过 Issues 提交理性技术探讨。

---

### 提交 Issue 说明

在提交 Issue 前，请确保已查阅以下规范：

1. **缺陷反馈 (Bug Report)**
   * 提供清晰复现步骤、操作系统版本与日志输出。
   * 说明期望行为与实际行为差异。

2. **功能建议 (Feature Request)**
   * 详细说明提议功能的具体场景与实际价值。
   * 保持理性的架构讨论，避免非必要的复杂化设计。

---

### Pull Request 流程

1. Fork 本仓库并新建功能分支。
2. 确保提交信息严格遵循 Commit 规范：`[前缀(修改对象): 精准改动摘要 (YYYY-MM-DD HH:mm:ss)]`。
3. 确保更改通过本地构建与语法检查。
4. 提交 Pull Request 并详细附带更改说明。
